Laborator 3

Selectorii de css

Data trecuta am vorbit pe scurt despre diferiti tipi de selectori de css precum clasa, tag si id

In limbajul css, fiecarui tip de selector ii e atribuit un simbol.

  • class: (.) + numele clasei
  • id: (#) + numele clasei
  • tag: doar numele tagului

Exemplu de declarat taguri cu clase si id-uri

<div id="comp-1" class="compartiment-principal compartiment"></div> <div id="comp-2" class="compartiment"></div> <div id="comp-3" class="compartiment"></div> <!-- inline-style defineste css pentru tagul acesta: prioritatea este maxima --> <div id="comp-4" style="background-color: aqua;" class="compartiment"></div>

Exemplu de limbaj css:

/* css pentru tag-ul cu id-ul comp-1 */ #comp-1{ background-color: red; } /* se aplica pe toate elementele care au in lista de clase: compartiment */ /* proritate mai mare decat tag*/ .compartiment{ background-color: blue; } /* se aplica pentru toate tagurile div*/ div { background-color: yellow; }

Cateva detalii despre selectorii principali:

  • Fiecare element poate avea unul sau mai multe clase
  • Clasele se despart unele de altele prin spatiu
  • Fiecare element poate avea un singur ID unic in tot fisierul HTML

Alt tip de selectori

  • "*" : Se refera la toate elementele
  • "x > y" : se refera la copii directi ai elementului / elementelor x, identificati prin y

    • Exemple

    /* toate paragrafele din elementele cu clasa container (un singur nivel) .container > p { color: red; }
  • "x y" : se refera la copii de orce grad al elementului / elementelor x, identificati prin y
    • /* toate paragrafele din elementele care se gasesc mai adanc in interiorul elementului container */ .container p { color: yellow; }

Prioritate:

Dupa cum probabil ati vazut in exemplu de mai sus, fiecare tip de selector are o anumita prioritate

  • Poti da overwrite la o definitie in css daca o definesti inca odata (o sa-si pastreze si definitiile de mai sus daca nu le-ai schimbat)
    • .compartiment{ background-color: blue; } /* background-ul o sa fie rosu */ .compartiment{ background-color: red; }
  • Nu conteaza in ce ordine sunt puse atributele in fisierul css. Prioritatea o sa fie urmatoarea inline-style > id > class > tag
  • Cu cat un selector este mai bine definit, cu atat prioritatea este mai mare. Vedeti mai specific cum se calculeaza aici
  • Daca chiar vreti ca nimeni sa nu va mai schimbe in viitor css-ul (nici macar voi), puteti folosi valuarea !important dupa un attribut de css. Acel atribut o sa aiba prioritate pana si peste inline-style. Exemplu /* Prioritar peste tot */ .compartiment { background-color: purple !important; } Acest tip de lucru ar trebuii sa fie cat mai evitat pentru a mentine proiectul curat si pentru a lasa oportunitatea si altor codeveloperi sa creeze teme noi si sa poata modifica stiluri deja existente.

Positionarea unui element

Exista mai multe tipuri de pozitionare. Principalele sunt

  • relative
  • absolute
  • fixed

Un element cu atributul in css: position: relative; este relativ la pozitia lui normala.

Initial pozitia lui nu o sa fie schimbata, dar acum poate fi modificata prin atributele: top, right, bottom, left ca in exemplu de mai jos

<div style="padding: 5px; width: 300px; height: 100px; border: solid 2px red;"> <div style="width: 30px; height: 30px; border: solid 1px;" id="element-simplu"></div> <!-- i-am atribuit position: negative, l-am deplasat fata de top cu 10px si fata de stanga cu 20px --> <!-- Puteti sa dati click dreapta pe al doilea patrat si sa testati felul in care se deplaseaza fata de pozitia lui initiala modificand atributul top si left --> <div style="width: 30px; height: 30px; border: solid 1px; position: relative; top: 10px; left: 30px;" id="element-pozitie-relativa"></div> </div>

Un element cu atributul in css: position: absolute; are pozitia relativa la primul lui ancestor cu pozitia relativa

Este util atunci cand vrei sa pozitionezi un button in colul din dreapta, sau o iconita pentru ai arata user-ului ce tip de fereasta a deschis, fara sa te mai ingrijorezi de cum o sa fie asezate restul componentelor datorita acestuia

<!-- De data asta div-ul container are pozitia relativa --> <div style="padding: 5px; width: 300px; height: 100px; border: solid 2px red; position: relative;"> <div style="width: 30px; height: 30px; border: solid 1px;"></div> <!-- i-am atribuit position: negative, l-am deplasat fata de top cu 10px si fata de stanga cu 20px --> <div style="width: 30px; height: 30px; border: solid 1px; position: relative; top: 10px; left: 30px;"> </div> <!-- elementul cu pozitie absoluta. Fiind o pozitie absoluta nu o sa tina cont de celelalte elemente. S-a asezat peste ele --> <div style="width: 30px; height: 30px; border: solid 2px green; position: absolute; top: 10px; left: 30px;"> </div> </div>

position: fixed; o sa pozitioneze elementul relativ la ecran. Puteti da click dreapta pe patratelul verde din chenarul rosu, inspect element si sa-i schimbati pozitia in fixed pentru a urmarii functionalitatea

Este util atunci cand vrei sa faci un meniu care sa urmareasca userul pe tot parcursul paginii sau o reclama enervanta pe care sa dea click din greseala, ati prins voi idea.
