Cerinte proiect

Exista 2 tipuri de cerinte :

  • Eliminatorii - nu se prezinta proiectul fara ele; Sunt notate cu rosu
  • Recomandate - va ajuta sa acumulati puncte; Puteti prezenta si fara ele

Punctaj toatal proiectul de Javascript -- 2 puncte

Generale - fara punctaj, dar sunt eliminatorii

  1. Codul pus pe github. Daca nu aveti deja un cont, trebuie sa va faceti unul (dureaza 5 minute si este gratuit). Github
  2. Cate un fisier JS extern pentru fiecare pagina. Puteti avea si un fisier comun, daca sunt functii care se repeta
  3. Validarea codului Javascript (sa nu aveti erori in consola; ex: numeVariabila is not defined)
  4. Sa adaugati comentarii in cod si sa aveti un cod "bine scris"(fara variable care se numesc a,b,c, codul care se repeta pus in functii etc).

    5. Gasiti pe internet mai multe ghiduri care explica standardele folosite in industrie, cele mai populare fiind cele introduse de GitHub, Google sau AirBnB.

AJAX si Node.js

  1. Realizarea de apeluri ajax catre un API creat in Node.js. → 0.15p
  2. Crearea unui API simplu in Node.js care implementeaza CRUD (Create, Read, Update, Delete). → 0.1 pt fiecare metoda( Create, Read, Update, Delete) Exemplu: Am o librarie online.
    • Create - adaugam o carte noua (nu trebuie salvata in baza de date, dar o putem salva intr-un fisier sau intr-un array pe server)
    • Read - citim toate cartile din fisier (inclusiv pe cele pe care le-am adaugat mai devreme)
    • Update - actualizam datele despre o anumita carte (ex: titlul, autorul, anul publicarii etc)
    • Delete - stergem o carte din fisier/array
    OBS: Obligatoriu sa aveti cel putin create si read; Update si delete sunt optionale, dar nu sunt grele
  3. Pagina pentru eroarea 404. → 0.1p Ex: Vreau sa intru pe https://aplicatiaMea/calculatorPensie , dar nu exista pagina "calculatorPensie" si imi apare o pagina stilizata pentru eroarea 404. Articol despre eroarea 404
